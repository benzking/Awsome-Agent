## 📋 分析前必读

在开始分析前，请先阅读项目 README（优先英文版 `README_EN.md`，若无则用 `README.md`），再阅读项目源码结构（特别是 `SKILL.md`/`CLAUDE.md`/`agent.py`/`main.py` 等入口文件）。

**抓取顺序建议：**
1. `README.md` / `README_EN.md` — 项目概述
2. 源码入口文件 — 了解实际功能
3. `package.json` / `setup.py` / `pyproject.toml` — 依赖和配置
4. 示例文件或 `example/` 目录 — 快速上手体验

### 三、应用标签分类参考

> 注：一个项目可能同时属于多个标签

| 标签 | 含义 | 典型场景 |
|------|------|---------|
| **Agent** | AI Agent 框架或应用 | 自动化任务执行、多步骤推理 |
| **Plugin** | 插件 / 扩展 | 集成到现有平台的功能增强 |
| **Skill** | Claude Code / OpenClaw 等 Agent 技能 | 特定能力的扩展命令 |
| **MCP** | Model Context Protocol 协议实现 | 标准化工具调用接口 |
| **Tool** | 工具库 / SDK | 提供 API 封装供开发者调用 |
| **应用** | 完整可用的终端产品 | SaaS / Desktop / CLI 应用 |
| **RAG** | 检索增强生成相关 | 知识库问答、文档检索 |
| **Knowledge** | 知识管理相关 | 知识图谱、本地笔记系统 |

#### 8.2 安装步骤

> 按顺序列出，每步附带命令

**方式一：npm 安装（推荐）**
```bash

```

**方式二：Git Clone + 本地安装**
```bash
git clone https://github.com/{owner}/{repo}.git
cd {repo}
pip install -r requirements.txt   # Python 项目
# 或
npm install && npm run build     # Node.js 项目
```

**方式三：Skill 安装（Claude Code / OpenClaw）**
```bash
npx skills add {owner}/{repo}
```
